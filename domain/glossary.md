## Entity Definitions

### User
```
Fields:
- id: UUID, unique identifier
- email: string, unique, validated
- name: string, display name
- role: enum (admin, member, guest)
- created_at: timestamp
- updated_at: timestamp

Relations:
- has many Projects
- belongs to Workspace
```

### Project
```
Fields:
- id: UUID, unique identifier
- name: string, display name
- description: text, optional
- status: enum (active, archived, draft)
- owner_id: UUID, foreign key to User
- workspace_id: UUID, foreign key to Workspace

Relations:
- belongs to User (owner)
- belongs to Workspace
- has many Tasks
```

## Status Definitions

| Status | Meaning | Transitions |
|--------|---------|-------------|
| draft | Not yet published | draft -> active |
| active | Currently in use | active -> archived |
| archived | Soft deleted, recoverable | archived -> active |
| deleted | Hard deleted | terminal state |

## Role Definitions

| Role | Permissions | Description |
|------|-------------|-------------|
| admin | full access | Can manage users, settings |
| member | read/write | Can create and edit content |
| guest | read only | Can view public content |
